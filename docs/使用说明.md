# 使用说明

## 日常使用流程

每周运行：

```bash
python scripts/run_market_scan.py --limit 800
python scripts/run_tracking.py
```

想看短期冒头基金：

```bash
python scripts/run_short_term_radar.py --limit 800
```

想复盘某个历史日期：

```bash
python scripts/run_time_machine.py --as-of YYYY-MM-DD --horizon 1m,3m,6m,12m --top-n 10 --limit 800
```

想验证新星、生命周期、资金迁移信号：

```bash
python scripts/run_v2_lite_validation.py --limit 800
```

想生成资金决策辅助计划：

```bash
python scripts/run_v3_full.py --limit 800
```

主要输出位置：

- `reports/<日期>/基金雷达扫描结果.xlsx`
- `reports/<日期>/分析摘要.md`
- `reports/<日期>/短期异动雷达.xlsx`
- `reports/v2_lite/<日期>/V2验证报告.xlsx`
- `reports/v3/<日期>/portfolio_allocation.xlsx`
- `reports/v3/<日期>/dca_plan.xlsx`
- `reports/v3/<日期>/exit_plan.xlsx`
- `reports/v3/<日期>/rotation_signal.xlsx`
- `reports/v3/<日期>/V3_backtest.xlsx`
- `reports/v3/<日期>/v3_summary.md`

## 短期异动雷达

```bash

```


## V3 资金决策辅助

```bash
python scripts/run_v3_full.py --limit 800
```

V3 只读取 V1/V1.1/V2-lite 输出，生成仓位、定投、止盈、轮动和回测报告。它不是预测系统，也不是荐基工具。

## V4.2 日常统一运行

日常运行不要分别手动执行旧脚本，优先使用统一入口：

```bash
python scripts/run_daily_all.py --limit 800
```

该命令会按顺序调度：

1. `market_scan`
2. `short_term_radar`
3. `v3_full`
4. `v3_tracker`
5. `v4_full`
6. `daily_report`

每一步都会写入 `reports/daily/YYYY-MM-DD/run_guard.json`。如果某一步已经完成、输出存在且最新净值日期未变化，默认会跳过。

常用命令：

```bash
python scripts/run_daily_all.py --status
python scripts/run_daily_all.py --step daily_report
python scripts/run_daily_all.py --step short_term_radar
python scripts/run_daily_all.py --report-only
python scripts/run_daily_all.py --limit 800 --force
```

`--force` 会强制重跑，可能耗时很久；常规日常使用不建议默认加 `--force`。

## Windows 自动运行

注册每日自动运行任务：

```powershell
.\scripts\register_fundradar_daily_task.ps1 -Time "23:30"
```

检查任务状态：

```powershell
.\scripts\check_fundradar_daily_task.ps1
```

删除任务：

```powershell
.\scripts\unregister_fundradar_daily_task.ps1
```

自动任务实际执行：

```powershell
python scripts/run_daily_all.py --limit 800
```

运行日志保存在：

```text
logs/daily_runner/
```

提醒：不要每天使用 `--force`。`--force` 会完整重跑 `market_scan`，可能耗时很久。

## 低冗余日常检查

数据健康检查只读取本地日志、缓存、日报 run_guard 和已有报告，不触发外部接口：

```bash
python scripts/run_health_check.py
```

输出：

```text
reports/health/YYYY-MM-DD/health_report.md
reports/health/YYYY-MM-DD/health_report.xlsx
```

构建日报历史索引：

```bash
python scripts/build_daily_index.py
```

输出：

```text
reports/daily/index.md
reports/daily/index.xlsx
```

这些工具只做信息压缩、健康检查和历史索引，不改变 V1 到 V4.3 的任何计算逻辑，也不新增预测或荐基功能。

## 盘中净值估算观察

下午 3 点前想看观察池和 V3 组合的盘中估算涨跌，可以运行：

```bash
python scripts/run_intraday_estimation.py
```

可选指定基金分类：

```bash
python scripts/run_intraday_estimation.py --categories 股票型,混合型 --top-n 30
```

输出：

```text
reports/intraday/YYYY-MM-DD/intraday_estimation.md
reports/intraday/YYYY-MM-DD/intraday_estimation.xlsx
```

注意：盘中估算净值不是最终公布净值，可能与晚间正式净值有偏差。该模块只用于人工观察，不构成买卖建议，也不改变 V1-V4 任何计算逻辑。
