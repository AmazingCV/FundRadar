# 报告目录说明

`reports/` 是运行输出目录，不进入 Git。当前采用“日常输出在根层、历史研究归档到子目录”的结构。

## 日常优先查看

| 目录 | 用途 | 是否日常查看 |
|---|---|---|
| `reports/daily/YYYY-MM-DD/` | 每日晨报，统一阅读入口 | 是 |
| `reports/health/YYYY-MM-DD/` | 数据健康检查 | 是 |
| `reports/v3/YYYY-MM-DD/` | V3 纸面资金执行模拟 | 按需 |
| `reports/v3_tracker/YYYY-MM/` | V3 纸面跟踪记录 | 按需 |
| `reports/v4/YYYY-MM-DD/` | 主题暴露与拥挤度观察 | 按需 |
| `reports/intraday/YYYY-MM-DD/` | 盘中净值估算观察 | 交易日盘中按需 |
| `reports/v1_market/YYYY-MM-DD/` | V1 中线全市场扫描，未来新输出路径 | 按需 |
| `reports/v1_1_short_term/YYYY-MM-DD/` | V1.1 短期异动雷达，未来新输出路径 | 按需 |

## 研究和历史归档

| 目录 | 用途 |
|---|---|
| `reports/research/backtest/` | 历史回测 |
| `reports/research/time_machine/` | 历史时点模拟 |
| `reports/research/weight_search/` | 中线权重搜索 |
| `reports/research/short_term_weight_search/` | 短期权重搜索 |
| `reports/research/stage_return_compare/` | 指定基金阶段收益对比 |
| `reports/legacy/` | 早期旧目录结构的历史输出 |

## 口径

- 日常只需要先看 `reports/daily/YYYY-MM-DD/daily_report.md`。
- `reports/health/YYYY-MM-DD/health_report.md` 用来判断当天数据是否可靠。
- `reports/research/` 用于复盘和验证，不作为每日阅读入口。
- `reports/legacy/` 只是历史兼容目录，代码读取端已做递归兼容。
