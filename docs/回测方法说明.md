# 回测方法说明

## 避免未来函数

每个扫描日只使用该日期及以前的净值数据计算：

- 近1月、近3月、近6月、近1年收益；
- 近1年最大回撤；
- 近1年波动率；
- 综合评分。

未来收益验证：

- 买入价格：扫描日当天或之后第一个可用净值；
- 卖出价格：目标日期当天或之前最近一个可用净值；
- 对比对象：同一扫描日全候选池平均收益。

## 时间机器

`time_machine.py` 支持指定过去日期，例如 `2025-06-18`。系统先生成当时观察池，再用之后真实数据验证未来 1/3/6/12 月收益、最大回撤、相对全候选池和沪深300的超额收益。

时间机器里的主线验证字段 `组合后验表现支持` 不是未来持仓验证，也不是主题本身的独立收益验证。它只是说明：当时识别出该主题的精选组合，在后续已到期窗口中是否跑赢全候选池或基准，只能作为后验支持信号。

## 留出验证

默认：

- 训练/调参区间：2023-01-01 到 2025-06-18；
- 留出验证区间：2025-06-18 到当前日期。

验证区间不参与权重选择。训练强、验证弱的方案会标记为疑似过拟合。

## 已知偏差

公开接口难以完整恢复历史基金存续池和历史分类，因此全市场回测可能存在幸存者偏差。结论只能说明规则是否有观察价值，不能说明可预测未来收益。
